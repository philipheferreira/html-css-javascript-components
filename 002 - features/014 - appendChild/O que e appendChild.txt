O que é o appendChild?
Pense no seu HTML como uma árvore genealógica. Cada tag (como <div>, <p>, <ul>) é um "pai" ou um "filho" de outra tag.

Um <div> que contém um <p> é o elemento pai.
O <p> que está dentro do <div> é o elemento filho.

O appendChild é um método (uma função que pertence a um objeto) em JavaScript que serve 
para adicionar um novo elemento filho ao final de um elemento pai já existente.

Em outras palavras: ele pega um elemento HTML que você criou (ou que já existe) e o 
"coloca" dentro de outro, como o último filho dele.



Pontos Importantes a Saber

1. Move, não Copia: Se o elementoFilho já existir em algum lugar no seu HTML, o 
appendChild irá movê-lo para a nova posição, em vez de criar uma cópia. Ele não
 pode existir em dois lugares ao mesmo tempo.

2. Sempre no Final: O appendChild sempre adiciona o novo elemento como o 
último filho. Se você precisar inserir um elemento em uma posição específica 
(por exemplo, no início), você usaria outros métodos como elementoPai.insertBefore().

3. Alternativa Moderna (append): Existe um método mais novo chamado append(). 
Ele é mais flexível porque pode adicionar múltiplos elementos e até mesmo strings de texto 
de uma vez. Exemplo: elementoPai.append('Texto simples', elementoFilho1, elementoFilho2);. 
No entanto, appendChild é ainda muito comum e fundamental de entender.

Resumo:

O appendChild é sua ferramenta principal para construir e modificar a página dinamicamente. 
Ele permite que seu JavaScript altere a estrutura do seu HTML em tempo real, sem precisar 
recarregar a página, criando experiências interativas para o usuário.
